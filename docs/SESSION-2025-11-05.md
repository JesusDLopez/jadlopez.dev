## ğŸ“ Key Learnings

### **For Future Development:**

1. **Case-Sensitivity Matters:**
   - Local development (Mac/Windows) is case-insensitive
   - Production (Vercel/Linux) is case-sensitive
   - Always match exact folder/file names
   - Test builds locally: `npm run build`

2. **Environment Variables:**
   - Prefix with `VITE_` for Vite projects
   - Set in Vercel Dashboard â†’ Settings â†’ Environment Variables
   - Requires redeploy after adding/changing
   - Use `.env.local` for local testing (gitignored)

3. **Deployment Workflow:**
   - Develop on feature branch
   - Push triggers preview deployment
   - Test preview before merging to main
   - Main branch auto-deploys to production

4. **Privacy Implementation:**
   - Free code-based solutions exist
   - Multiple layers provide redundancy
   - Easy to enable/disable via env vars
   - Can be completely removed later
